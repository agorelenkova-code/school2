### Алгоритм

Для каждой таблицы выполни следующие шаги:

**Шаг 1 — получи метаданные:**
```bash
~/.local/node_modules/@googleworkspace/cli/bin/gws sheets spreadsheets get \
  --params '{"spreadsheetId":"<ID>","fields":"properties(title),sheets(properties(sheetId,title,gridProperties))"}'
```

**Шаг 2 — считай заголовки и первые строки:**
```bash
~/.local/node_modules/@googleworkspace/cli/bin/gws sheets spreadsheets values get \
  --params '{"spreadsheetId":"<ID>","range":"Sheet1!A1:Z6"}'
```

**Шаг 3 — зафикcируй:**
- название таблицы (`properties.title`)
- URL: `https://docs.google.com/spreadsheets/d/<ID>`
- Spreadsheet ID
- Лист: `Sheet1`, GID: `0`
- заголовки колонок (строка 1)
- содержимое первых 3–5 строк данных

**Шаг 4 — опиши таблицу по шаблону ниже**

### Формат sheets-index.md

```markdown
# Индекс Google Sheets
_Обновлено: YYYY-MM-DD_

---

## Таблица N: {название таблицы}

URL:
https://docs.google.com/spreadsheets/d/...

Spreadsheet ID:
...

Лист:
Sheet1

GID:
0

Ключевые колонки:
- Колонка A: название — тип данных
- ...

Что лежит в таблице:
[Конкретное описание: кто, что, за какой период, в каком формате]

Ключевые сущности:
- ученики / семьи / классы / кураторы / родители / статусы / ...

Для каких вопросов использовать:
- [конкретные задачи]
- Количество учеников у ответственного по классу: да / нет / частично
- Многодетные семьи 3+ детей: да / нет / частично
- Кто ведёт конкретную семью: да / нет / частично

Как искать:
- Фамилия ученика: колонка X
- Семья: колонка Y
- Класс: колонка Z
- Ответственный/куратор: колонка Q
- Количество детей: колонка / поле / не применимо

Ограничения:
- [конкретные ограничения именно этой таблицы]

Шаблон ссылки на строку:
https://docs.google.com/spreadsheets/d/{SPREADSHEET_ID}/edit?gid=0&range=A{ROW}:Z{ROW}

---
```

### Правила

- **Читай только** — никаких изменений в таблицах.
- Если таблица недоступна — добавь запись с пометкой `⚠️ Недоступна (403/404)`.
- Описание «Что лежит в таблице» — только из реальных данных, не придумывай.
- Раздел «Для каких вопросов» должен явно отвечать на три стандартных вопроса: ученики у ответственного по классу / многодетные семьи / кто ведёт семью.
- После записи файла выведи краткую сводку: сколько таблиц добавлено, какие ограничения критичны.

### Особые ограничения — обязательно включить в индекс

**Таблица 3 — "Заметки — Павлов":**
- имена в формате «Фамилия И.» — соединение с другими таблицами по полному ФИО невозможно;
- поиск только по фамилии, риск коллизий у однофамильцев.

**Таблица 5 — "Семьи — Белов":**
- дети записаны в одном поле через `;` (пример: `Даниил (3А, Мини); Макар (6В, Стандарт)`);
- подсчёт детей — по количеству элементов через `;` или по колонке `Кол-во детей`, если есть;
- есть отдельная колонка `Кол-во детей` — использовать её в приоритете для фильтра 3+.

**Таблица 8 — "Родители — Тишина":**
- нет колонки с учеником — только фамилия семьи;
- для привязки к конкретному ученику нужна дополнительная таблица (напр. Таблица 1 или 10).

## Когда пересобирать индекс

Индекс пересобирается **вручную** командой `/sheets-index`. Запускай в следующих случаях:

- Добавлена новая Google Sheet как источник данных
- Изменилась структура колонок в существующей таблице
- Переименована таблица или лист
- `/sheets-search` возвращает ошибку «таблица не найдена» или устаревшие данные
- Планово — раз в месяц

> ⚠️ Автоматического обновления нет. Индекс отражает состояние таблиц на момент последнего ручного запуска `/sheets-index` (дата указана в первой строке `sheets-index.md`).
