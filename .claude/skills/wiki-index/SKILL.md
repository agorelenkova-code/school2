# Skill: /wiki-index

Обходит корпоративную вики и обновляет файл `wiki-index.md` в корне проекта — актуальный каталог всех страниц с заголовками, описаниями и ссылками.

## Prompt

Твоя задача — обойти корпоративную вики https://os1.lms.robotsatwork.ai/wiki/ и обновить файл `wiki-index.md` в корне проекта.

### Алгоритм

1. Загрузи главную страницу вики и найди все разделы верхнего уровня.
2. Для каждого раздела — загрузи страницу раздела и найди все подстраницы.
3. Для каждой подстраницы — загрузи страницу и извлеки:
   - Заголовок страницы
   - Короткое описание (1 предложение — первый абзац или аннотация)
   - URL
4. Сформируй структурированный Markdown-файл и запиши его в `wiki-index.md`.

### Формат wiki-index.md

```markdown
# Wiki Index
_Обновлено: YYYY-MM-DD_

## Раздел: [Название раздела]
- **[Заголовок страницы]** — [краткое описание]
  `[URL]`

## Раздел: [Название раздела]
...
```

### Правила

- Не пропускай разделы — обойди все, включая HR, Финансы, Продажи, Продукт, Техотдел, Общее.
- Если страница недоступна (404) — пропусти, не добавляй в индекс.
- Описание — только из содержимого страницы, не придумывай.
- Если описания нет — пиши `«описание отсутствует»`.
- После записи файла выведи краткую сводку: сколько разделов и страниц проиндексировано.

---

## Когда пересобирать индекс

Индекс пересобирается **вручную** командой `/wiki-index`. Запускай в следующих случаях:

- **После изменений в вики** — добавлена, удалена или переименована страница
- **После обновления регламентов** — изменилось содержание важных документов (инструкции, правила, скрипты)
- **Если `/wiki-search` даёт устаревший ответ** — нашёл страницу, которой уже нет, или не нашёл новый документ
- **Планово — раз в неделю**, чтобы индекс не расходился с реальным состоянием вики

> ⚠️ В текущей версии автоматического обновления нет: cron-задача и webhook не настроены. Индекс отражает состояние вики на момент последнего ручного запуска `/wiki-index` (дата указана в первой строке `wiki-index.md`).
