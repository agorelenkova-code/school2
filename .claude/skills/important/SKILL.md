# Skill: /important

Генерирует ежедневный дайджест для директора онлайн-школы «Ромашка» на основе сообщений из Discord.

## Использование

```
/important
/important --date 2026-04-15
```

- **Без аргументов** — дайджест за последние 24 часа
- **`--date YYYY-MM-DD`** — дайджест за конкретный день

---

## Что делать при вызове

### 1. Загрузить переменные окружения

Прочитай файл `~/school2/.env` и извлеки значение `DISCORD_BOT_TOKEN`.

### 2. Загрузить критерии фильтрации

Прочитай файл `~/school2/filter-criteria.md` — он определяет что важно, что игнорировать, жёсткие триггеры и формат вывода.

### 3. Получить сообщения из Discord

Выполни запросы к Discord API для каждого из четырёх каналов:

| Канал | ID |
|---|---|
| `#кураторы` | `1490709891721986219` |
| `#маркетинг` | `1490709924370452500` |
| `#техподдержка` | `1490709961557278792` |
| `#продажи` | `1490709980351828079` |

Запрос (подставить channel_id):
```bash
curl -s -H "Authorization: Bot $DISCORD_BOT_TOKEN" \
  "https://discord.com/api/v10/channels/{channel_id}/messages?limit=100"
```

Фильтрация по времени:
- Без аргументов: сообщения за последние 24 часа (от текущего момента)
- С `--date`: сообщения за указанные сутки (00:00–23:59 UTC)

Игнорировать системные сообщения (type ≠ 0).

### 4. Проанализировать сообщения по критериям

Для каждого канала сгруппируй сообщения по темам (тред или связанная цепочка). Применяй критерии из `filter-criteria.md`:

**Жёсткие триггеры — включать всегда:**
- Удаление персональных данных
- Технический сбой во время уроков
- Невыполнение плана продаж
- Судебные риски, иск, юридические претензии
- Негативные отзывы

**Включать если:**
- Тема не имеет финального решения (нет «ок», «закрыто», «сделано», «договорились»)
- Признаки незакрытости: поднята проблема без ответа, задан вопрос без ответа, есть риск без плана
- Обсуждение с 3+ участниками вокруг одной проблемы
- Эмоциональные высказывания, напряжение
- Упоминание невыполнения показателей или отклонений от плана

**Не включать:**
- Бытовые разговоры, неформальный чат
- Решённые вопросы без рисков
- Технические мелочи, устранённые в рамках одного обсуждения

### 5. Сформировать дайджест

Формат каждого пункта:

```
[#канал] Заголовок проблемы — краткое описание:
кто участвует, в чём суть, сколько дней висит (если это повтор),
есть ли решение. Если нет — явно написать «решения пока нет».
```

Структура дайджеста:

```
# Дайджест — [дата]

## 🔴 Требуют внимания директора
[Пункты с жёсткими триггерами или критическими незакрытыми проблемами]

## 🟡 Открытые вопросы
[Незакрытые темы без критического уровня]

## ✅ Закрыто за день
[Только если есть важные вещи, которые успешно решились — одной строкой каждая]

---
Каналы: #кураторы, #маркетинг, #техподдержка, #продажи
Период: [временной диапазон]
Сообщений обработано: [N]
```

Если нет ни одного важного события — написать:
```
# Дайджест — [дата]
Важных событий за период не обнаружено.
```

### 6. Вывести дайджест

Выведи дайджест в ответе. Не сохраняй в файл если пользователь не попросил.

---

## Правила

- Писать конкретно: не «обсуждали конверсию», а «конверсия лендинга 15 записей за 4 дня при ожидаемых ~30»
- Имена использовать как в Discord (Олег, Костя, Ира и т.д.)
- Если проблема повторяется из предыдущих дней — указать сколько дней висит
- Не интерпретировать и не давать советов — только факты из сообщений
- Секция «✅ Закрыто» опциональна — включать только если есть что-то существенное
