# Skill: meeting-analysis

Анализирует транскрипты созвонов из папки `calls/` и сохраняет структурированный отчёт в `report.md`.

## Использование

```
/meeting-analysis
/meeting-analysis calls/2026-05-15_cross-team-sync.md
/meeting-analysis --summary-only
```

---

## Что делать при вызове

### 1. Разбор аргументов

- **Без аргументов** — обрабатывать только новые файлы из `calls/*.md`, затем обновить сводный анализ.
- **Путь к файлу** — обработать только указанный файл (даже если он уже есть в report.md — перезаписать его секцию), затем обновить сводный анализ.
- **`--summary-only`** — не читать транскрипты, только пересчитать и перезаписать сводный анализ по уже существующим секциям в report.md.

### 2. Определение новых файлов

Открой `report.md` (если файл существует). Файл считается **уже обработанным**, если в report.md есть строка вида:

```
## 📋 [имя_файла.md]
```

Например: `## 📋 2026-05-05_weekly-sync.md`

Все файлы из `calls/*.md`, для которых такой заголовок не найден — считаются новыми и подлежат анализу.

### 3. Анализ каждого нового транскрипта

Прочитай файл полностью. Для каждого транскрипта извлеки четыре категории:

#### Ключевые решения
Что было принято как решение (утверждено, подтверждено, одобрено). Для каждого:
- Что именно решено
- Кто принял / инициировал
- Контекст (зачем, при каких обстоятельствах)

#### Открытые вопросы
Что обсуждалось, но осталось без ответа или отложено. Для каждого:
- В чём вопрос
- Кто поднял
- Почему завис (ждут человека, данных, решения)

#### Проблемы
Сложности, конфликты, блокеры, жалобы — всё что мешает работе. Для каждого:
- В чём проблема
- Кто пострадал / упомянул
- Контекст и последствия

#### Задачи с ответственными
Конкретные поручения с именем ответственного и дедлайном (если назван). Формат:
- `[Имя]` — что сделать — срок (если есть)

### 4. Запись в report.md

**Если report.md не существует** — создать файл с заголовком:

```markdown
# Анализ созвонов — Онлайн-школа «Ромашка»
```

**Для каждого обработанного файла** добавить секцию в конец файла (перед сводным анализом, если он есть):

```markdown
---

## 📋 [имя_файла.md]
**Дата:** [дата из транскрипта] | **Участники:** [список] | **Длительность:** [длительность]

### Ключевые решения
- **[Что решено]** — *[Кто]* — [контекст]

### Открытые вопросы
- **[Вопрос]** — *[Кто поднял]* — [почему завис]

### Проблемы
- **[Проблема]** — *[Кто упомянул]* — [контекст]

### Задачи
- **[Имя]** — [что сделать] — [срок]
```

Если в категории нет пунктов — пропустить её заголовок.

### 5. Обновление сводного анализа

После записи всех новых секций — найди и **полностью перезапиши** блок сводного анализа в report.md. Этот блок начинается с маркера `<!-- SUMMARY_START -->` и заканчивается `<!-- SUMMARY_END -->`. Если маркеров нет — добавить блок в самый конец файла.

Сводный анализ строится **по всем секциям** в report.md (не только новым).

Формат сводного анализа:

```markdown
<!-- SUMMARY_START -->
---

## 🔍 Сводный анализ повторяющихся паттернов

> Охват: [N] созвонов ([перечень файлов через запятую])
> Обновлено: [дата]

### Топ повторяющихся проблем

Для каждой из 3 наиболее частых проблем:

#### [Название проблемы]
- **Что не работает:** [описание]
- **Кто страдает:** [имена / роли]
- **Что теряет команда:** [последствия]
- **В каких созвонах:** [список файлов с кратким пояснением]

### Нерешённые вопросы (висят дольше всего)

Топ-5 вопросов, которые встречались в нескольких созвонах и до сих пор без решения:
- **[Вопрос]** — поднимался в: [файлы]

### Задачи без дедлайна или ответственного

Задачи из всех созвонов, у которых не назначен ответственный или не указан срок:
- [Описание задачи] — [из какого файла]
<!-- SUMMARY_END -->
```

---

## Правила

- Всё на **русском языке**.
- Имена участников использовать так, как они названы в транскрипте (Георгий, Лена, Алексей и т.д.).
- Быть конкретным: не «обсуждали сроки», а «Алексей обещал описания программ к пятнице 16 мая».
- Не добавлять оценочных суждений — только факты из транскрипта.
- Если в транскрипте дедлайн указан относительно («на этой неделе», «к среде»), конвертировать в абсолютную дату, исходя из даты созвона в заголовке файла.
- При `--summary-only` не читать транскрипты и не изменять секции отдельных файлов — только пересчитать сводный блок.
